# ウェイトリストLP ビジュアル強化 設計ドキュメント

- 日付: 2026-07-05
- ステータス: 承認済み
- 目的: `/lp` のビジュアルを「温かいフレンドリー」トーンで魅力的にし、コンバージョン（ウェイトリスト登録）を高める
- 前提: `docs/superpowers/specs/2026-07-05-waitlist-lp-design.md` のLPが実装済み（PR #84のブランチ上に追加コミットする）

## 決定事項

| 項目 | 決定 |
|------|------|
| トーン | 温かいフレンドリー（柔らかいグラデーション・丸み・暖色アクセント） |
| アセット | コード生成（SVG/CSS）のみ。画像ファイル・外部フォントは追加しない |
| 主役ビジュアル | 「山分けフロー図」インフォグラフィック（新規） |
| コピー | 既存の見出し・本文・フォーム文言は一切変更しない（テスト互換のため） |

## 1. カラー & トーン

- 既存のprimary（紫系）はCTA・強調に残す。LPの背景・装飾・アイコンチップに暖色（Tailwindの `amber` / `orange` / `rose` の淡いトーン）を追加する
- 実現手段はTailwindユーティリティクラスのみ。`globals.css` やテーマ変数は変更しない（LP外への影響ゼロ）
- ダークモードは `dark:` バリアントで両対応（例: `bg-amber-50 dark:bg-amber-950/30`）
- セクション背景はホワイト/暖色ソフト背景を交互に配置してリズムを作る（既存の `bg-muted/50` 交互構成を暖色系に置き換え）

## 2. 山分けフロー図（新規・主役）

`src/features/waitlist-lp/components/yamawake-flow.tsx`（サーバーコンポーネント、propsなし）

固定の金額例で「収入合算 → 経費控除 → 山分け」を図解する3ステップのインフォグラフィック:

1. **ふたりの収入をひとつに**: 「あなた ¥300,000」「パートナー ¥200,000」の2本のバーが合流して「¥500,000」に
2. **共通経費を引いて**: 経費ブロック「− ¥180,000」が差し引かれる
3. **残りを山分け**: 残り「¥320,000」が「¥160,000 / ¥160,000」に2等分され、中央にハートアイコン

- 実装はSVG + Tailwind（バーはdiv+flexでも可、矢印・合流線はSVG）
- レイアウト: デスクトップ（sm以上）は横並び3ステップ、モバイルは縦積み
- アクセシビリティ: 図全体に `role="img"` + 日本語 `aria-label`（例:「ふたりの収入50万円から共通経費18万円を引き、残り32万円を16万円ずつ山分けする図」）。ステップ見出しはテキストとして読める構造にする
- 配色はdatavizスキルの基準に従う（系列色のコントラスト、ラベルは色以外でも区別可能に）
- 配置: ヒーロー直下（シミュレーターの上）。index.tsxの合成順は Hero → **YamawakeFlow** → SettlementSimulator → 以下既存どおり
- 実装確定時の逸脱（承認済み計画 `docs/superpowers/plans/2026-07-05-lp-visual.md` で確定）: レイアウトは横並び3ステップではなく全幅バー3本の縦積み（モバイル/デスクトップ共通）、`role="img"` + aria-labelは図全体に1つではなくバー1本ごとに付与（AT向けにステップ単位で完結した説明になり改善）、矢印・合流のSVGコネクタは不採用（ステップ番号バッジで代替）

## 3. ヒーロー強化

- 背景: 暖色の柔らかいグラデーション（例: `bg-gradient-to-b from-amber-50 via-rose-50/50 to-background` + dark対応）と、`blur-3xl` のぼかしブロブ装飾2個（`aria-hidden`、`pointer-events-none`）
- バッジ: 見出し上に「先行登録で6ヶ月無料」のpill型バッジ（amber系）
- タイポ: 見出しの「山分け」部分をprimary〜roseのグラデーションテキストで強調（`bg-clip-text`）
- CTAボタンは既存のまま（primary、サイズlg）

## 4. セクション磨き込み（アイコンはlucide-react、導入済み依存のみ）

| セクション | 変更 |
|-----------|------|
| 課題共感 | 各カードに `FileSpreadsheet` / `ReceiptText` / `Scale` アイコンの暖色チップ（丸背景）を追加。カードは `rounded-2xl` + ソフトシャドウ |
| チーム思想 | 各項目に `ShieldCheck`（病気・事故）/ `Baby`（子育て）/ `HeartHandshake`（対等）アイコン。セクション背景を暖色ソフト帯に |
| 機能紹介 | `CalendarDays` / `ArrowRightLeft` / `Calculator` アイコンチップを追加 |
| 価格+フォーム | 価格部分をカード化（`rounded-2xl border`）し、「先行登録6ヶ月無料」バッジを添える。フォーム自体は変更しない |
| シミュレーター | カード枠を暖色ボーダーに、結果ボックスを淡いグラデーション背景に（軽い底上げのみ。ロジック・ラベル不変） |
| FAQ・フッター | 現状維持（背景色の交互リズム調整のみ可） |

## 5. 変更しないもの

- すべてのテキストコピー（見出し・本文・ラベル・ボタン文言・エラー/完了メッセージ）
- フォームのフィールド構成・Server Action・シミュレーターの計算ロジック
- `globals.css` / テーマ変数 / LP以外のページ
- 画像ファイルの追加（public/への配置なし）、外部フォント、追加npm依存

## 6. テスト

- 既存のコンポーネントテスト・E2Eはコピー不変のため無修正で通ることが完了条件
- 新規: `yamawake-flow` のスモークテスト1件（ステップ見出し3つと金額例の表示、`role="img"` のaria-label存在）
- 検証: CLAUDE.mdのUI検証ワークフローに従い、`npm run dev:mock` + Playwrightで 375px / 1280px × ライト / ダークの4パターンのスクリーンショットを撮り、表示崩れを目視確認する

## 7. やらないこと（YAGNI）

- シミュレーター入力と連動する動的フロー図（将来の拡張余地としてコンポーネントは独立させておく）
- アニメーションライブラリの追加（既存のmotion環境を超える演出）
- OGP画像の生成（別タスク）
- ライト/ダーク以外のテーマ対応
